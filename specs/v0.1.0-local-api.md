# v0.1.0 Local API

## 目的

`v0.1.0` では、ローカル環境でAgentログAPIを動かす。
KubernetesやGCPへ進む前に、APIの入出力とログ受信の最小機能を固める。

## 範囲

このMilestoneでは、FastAPIで次のAPIを実装する。

| Method | Path | 目的 |
|---|---|---|
| GET | `/health` | APIの生存確認 |
| POST | `/logs` | Agent操作ログの受信 |
| GET | `/logs` | 受信済みログの確認 |

保存先はメモリにする。
永続化、認証、Pub/Sub、BigQueryは扱わない。

## ログスキーマ

MVPでは次のフィールドを受け取る。

| Field | Type | Required | 説明 |
|---|---|---|---|
| `agent_id` | string | yes | Agent識別子 |
| `actor_user_id` | string | no | Agentを利用した人間ユーザー |
| `tool_name` | string | yes | 利用ツール名 |
| `action` | string | yes | 実行アクション |
| `status` | string | yes | `success` または `failure` |

## 受け入れ条件

`GET /health` は、HTTP 200と次のJSONを返す。

```json
{
  "status": "ok"
}
```

`POST /logs` は、正しいJSONを受け取り、受信内容をメモリに保存する。
レスポンスには保存したログを識別できる情報を含める。

`GET /logs` は、受信済みログの配列を返す。
プロセス再起動でログが消えることは、このMilestoneでは許容する。

不正なJSONまたは必須フィールド不足では、FastAPIのvalidation errorを返す。

## 確認方法

ローカルでAPIを起動する。

```sh
cd services/agent-log-api
uvicorn app.main:app --reload
```

ヘルスチェックを確認する。

```sh
curl http://localhost:8000/health
```

ログを登録する。

```sh
curl -X POST http://localhost:8000/logs \
  -H "Content-Type: application/json" \
  -d '{"agent_id":"agent-001","actor_user_id":"user-001@example.com","tool_name":"gmail.search","action":"read_email","status":"success"}'
```

ログ一覧を確認する。

```sh
curl http://localhost:8000/logs
```
